1. これは何を作るもの？

このプロジェクトは、介護福祉士試験の対策用「4択クイズ動画」（縦型・スマホ向け／Instagramリール等）を作る仕組みです。 1本の動画には 3〜4問 の4択クイズが入り、問題→考える時間→答え→解説、という流れでキャラクターが進行します。

すでに 動く見本 が入っています。まずはこれを開いて「こういうものを作るんだ」とイメージをつかんでください。

見本の場所datas/kaigo/ninchisho_rikai/four_choice/（科目：認知症の理解）
できあがる動画同じフォルダの 01-output.mp4, 02-output.mp4
迷ったらこの見本をマネする 新しい科目を作るときも、この ninchisho_rikai フォルダと同じ形をマネすれば失敗しません。 分からなくなったら「ninchisho_rikai の見本と同じようにして」とClaudeに言えばOKです。

4. ★ いちばんの仕事：問題チェック

ここが今回いちばんやってほしい作業です。 1問ずつ、下のチェックリストに沿って見て、直したいところがあれば Claude Code に指示して直してもらう——これだけです。

見るファイルは「.tsx」です（.md ではありません） チェック・修正の対象は、各科目フォルダの中の NN-FourChoiceData.tsx（例：01-FourChoiceData.tsx）です。
.md ファイル（ichimondittou.md / qa.md）は今回は見なくてOKです。
チェックするファイルdatas/kaigo/ninchisho_rikai/four_choice/01-FourChoiceData.tsx など（02-…03-…も同様）
1ファイル動画1本ぶん（4択クイズが3〜4問）
直し方自分で書き換えない。Claude Code に「ここを直して」と指示する

ファイルの読み方（1問のかたまり）

ファイルを開くと、1問が次のようなかたまりで並んでいます。見るのは 4か所だけです。

{
    question: '認知症のBPSDの特徴は?',                         ← ①問題文
    choices: ['認知症患者に必ず出現する',                       ← ②選択肢（4つ）
              '環境やケアが要因となり現れる',
              '中核症状に分類される',
              '進行に伴い必ず重度化する'],
    correctIndex: 1,        ← ③正解の番号（0=1つ目, 1=2つ目, 2=3つ目, 3=4つ目）
    explanation: 'BPSDは…適切な対応で改善も期待できる。',       ← ④解説
    questionAudio: ...  questionDuration: ...                    ← ※音声用。今は見なくてOK
}
「correctIndex」だけ注意 正解は correctIndex の数字で決まります。0=1つ目／1=2つ目／2=3つ目／3=4つ目です。 上の例は correctIndex: 1 なので、正解は2つ目の「環境やケアが要因となり現れる」です。

チェックリスト（1問ごとに上から確認）

各問題（question から explanation までのかたまり）ごとに、次を上から見ていきます。 1つでも「あやしい」と思ったら、その問題を Claude Code に直してもらいます。

  • 問題文は短い？　パッと一目で読める長さか（だらだら2〜3文になっていないか） ▶ 長い・説明的なら「もっと短く」
  • 選択肢4つの長さのバランスは崩れてない？　1つだけ突出して長い／短いものがないか（4つとも同じくらいなら、長め・短めどちらでもOK） ▶ バランスが悪ければ「長さをそろえて」
  • 正解だけ目立ってない？　正解だけ長い／正解だけ丁寧な言い回し、になっていないか ▶ これがあると知識なしでも正解が“バレる”。一番ありがちなNG
  • 消去法で解けてしまわない？　誤答3つが明らかにありえない／「必ず・絶対」で簡単に切れる、になっていないか ▶ あれば「誤答ももっともらしく直して」
  • 答えが問題文に透けてない？　問題文の中に答えがそのまま書かれていないか／選択肢に同じ意味のものが2つないか ▶ あれば「答えが透けないよう問題文か選択肢を直して」
  • 解説が長すぎない？　1〜2文くらいに収まっているか ▶ 長ければ「解説を短くまとめて」
いちばん大事なのは上から3番目 「正解だけ長い・正解だけ言い回しが丁寧」は、介護を知らない人でも当てられてしまう典型パターンです。 4つの選択肢が同じくらいの長さ・同じ調子でそろっているか、を特に意識して見てください。

直したいときは Claude Code に指示する

自分でファイルを書き換える必要はありません。どのファイルの何問目を、どう直したいかを Claude Code に伝えるだけでOKです。

「01-FourChoiceData.tsx の3問目、選択肢が長すぎる。短くそろえて。」
「01-FourChoiceData.tsx の2問目、正解だけ言い回しが丁寧で知識なしでも当てられる。誤答ももっともらしく直して。」
「01-FourChoiceData.tsx の1問目、correctIndex が答えと合っていない気がする。確認して直して。」
迷ったら丸ごと頼んでOK どこが悪いか言葉にしづらいときは「01-FourChoiceData.tsx の問題を全部チェックして、上のチェックリスト基準で直して」とまとめて指示しても大丈夫です。 品質ルールはClaude Code側に登録済みです。

6. 最新を取り込む（pull）／保存する（push）

このプロジェクトは GitHub（github.com/isihiro87/hoiku）でみんなと共有しています。 作業の前に「最新を取り込む（pull）」、作業の後に「保存する（push）」のが基本の流れです。

① 作業を始める前に：最新を取り込む（pull）

ほかの人が直した内容を取り込んでから始めます。これをしないと、古い状態のまま作業してしまいます。

方法A：Claude Code に頼む（かんたん）

「GitHub から最新を pull して。」

方法B：VS Code の画面で操作する

  1. 左の ブランチ／同期マーク（画面左下に main と表示されている所）をクリック、または
  2. 左メニューの ソース管理（枝アイコン）→ 上の「…」→ 「プル（Pull）」を選ぶ

方法C：ターミナルで打つ

git pull
pull のときに赤いメッセージが出たら 自分の手元の変更とぶつかると、警告が出て止まることがあります。 そのときは無理に進めず、メッセージをそのまま Claude Code に貼って「これどうすればいい？」と聞いてください。 安全に解決してくれます。

② 作業が終わったら：保存する（push）

チェックや修正が終わったら、内容を保存して GitHub に送ります（他のパソコン・他の人にも反映されます）。

「今日直した分を git に保存（コミット）して push して。」
知っておくと安心 ・「pull＝最新を受け取る」「push＝自分の変更を送る」とおぼえればOK。
・毎回、始める前に pull、終わったら push。これだけで他の人とぶつかりにくくなります。
・別PCで初めて始めるときは、git clone https://github.com/isihiro87/hoiku.git → フォルダを開く → npm install（→ 3章）。
・chrome-headless-shell/ という重いフォルダはコピー不要です（無くても自動でダウンロードして動きます）。

7. 困ったときは（よくある質問）

Q. 何から手をつければいい？

datas/kaigo/ninchisho_rikai/four_choice/01-FourChoiceData.tsx を開いて、4章の3つのポイントで上から1問ずつチェックするのが最初の仕事です。分からない行はそのままClaudeに貼って「これどういう意味？」と聞いてOKです。

Q. コマンドや専門用語が分からない

覚えなくて大丈夫です。やりたいことを日本語でClaudeに言うのが正攻法です。例：「動画を作りたい」「音声を入れ終わった」「保存して」。

Q. 問題を1問だけ直したい（もう一部は完成している）

「01の3問目だけ直したい。ほかは触らないで」と伝えてください。投稿済み・完成済みの動画を守る手順（ロック）がスキルに用意されています。誤答だけの修正なら音声の録り直しも不要です。

Q. 音声を録り直すと、ほかの番号もずれない？

直した内容録り直しが必要な音声
問題文を変えたその問題の音声だけ
正解の選択肢の文を変えたその答えの音声だけ
誤答（不正解の選択肢）だけ変えた不要（音声は変わらない）

Q. 新しい科目（例：「こころとからだのしくみ」など）を追加したい

新しい科目には専用の色（テーマ）を1つ追加する必要があります。これは少しコードを触るので、 Claudeに まるごとお願いしてください。

「介護福祉士の新しい科目『〇〇』を追加したい。科目スラッグと色（SubjectTheme）を追加して、 datas/kaigo/〇〇/four_choice/ を作って。ninchisho_rikai と同じ作りで。」

色の設定は src/components/rhythm/IchimonIttoBoard.tsx にまとまっており、 見本の ninchisho_rikai（深紫）がすでに追加済みなので、それをお手本に増やせます。

Q. レンダリングが失敗する／エラーが出た

エラーの文をそのままコピーしてClaudeに貼ってください。よくある原因は「音声ファイルの名前・番号ズレ」「audiosに足りない音声がある」です。

命令ファイル（render.sh）について NN-render.sh は以前のコンテナ環境向けのパスで書かれていることがあります。 そのまま動かすと失敗する場合があるので、レンダリングは「このPCの環境でレンダリングして」とClaudeに頼むのが確実です。 Claudeがこのパソコンに合わせて実行します。
