# コンテンツ修正規則（保育士資格試験クイズ）

ユーザーとのレビューで確立した修正基準。**保育士試験**の問題コンテンツに適用する。

適用対象（本プロジェクトのデータ）:

- **4択クイズ**: `datas/hoiku/[科目]/four_choice/NN-FourChoiceData.tsx` の `fourChoiceData[]`
  （編集対象フィールドは **`question` / `choices` / `correctIndex` / `explanation`** のみ）
- **○×一問一答**: `datas/hoiku/[科目]/[NN_トピック]/quiz/qas.md`（問題／答え／補足）

> **`question` / `choices` / `correctIndex` / `explanation`（および qas.md の問題／答え／補足）以外のフィールドは変更しない。**
> `id` / `questionAudio` / `answerAudio` / `questionDuration` / `answerDuration` は触らない。

> **音声との整合に注意**: 問題文（`question`）を変えたら**質問音声**、正答の選択肢の文言を変えたら**解説音声（answerAudio）**の再録が必要。誤答（distractor）だけの変更なら音声は再利用可。詳細は `SKILL.md`「質問の修正と音声の再録判断」。

> **本プロジェクトに純粋なフラッシュカード（front=答え/back=問い）は無い。** §A はその形式を扱う場合の規則として保持しつつ、○×一問一答（問い・補足）の品質確認にも援用する。
> 以下の具体例の一部は流用元（中学理科）プロジェクト由来の**説明用サンプル**。原則は資格試験でも共通だが、実適用時は保育士試験の内容に読み替える。

## 0. 資格試験向け4択問題の作り方ポイント（早見・要点）

詳細は §B。まずこの7点を満たすこと（カッコ内は実レビューで直した例）。

1. **問題文に答えを書かない／透けさせない。** 答えの語・略称・定義的特徴を本文に入れない。
   - 例: 「電気には＋と−の何種類？」→ 答え「2種類」がバレる → 「電気には何種類？」。
   - 例: 「DNAは何文字で表される？」のように問いに答え（DNA）が入る循環問題は、DNAが答えになる形へ。
2. **答えが一意に定まる問いにする。** 「どうなる？」など複数解が出る問いは条件で限定するか削除。
3. **4択は長さ・粒度・カテゴリ・形式をそろえる。** 正答だけ長い／具体的／複合（「AやB」）にしない＝それだけで答えがバレる。
   - 例: 正答「放射線や紫外線」だけ複合で長い → 「紫外線」に（他は単語）。
   - 例: 選択肢の冗長な共通語尾は削り、名詞で統一（問題文側で「何の○○」と受ける）。
4. **誤答（ディストラクタ）は同分野のもっともらしい誤りにする。** 別分野・ありえない・露骨に変な選択肢、「だけ/のみ」「すべて/絶対」等の手がかり語は禁止。正答位置はばらす。
5. **問題文・選択肢はコンパクトに。** 前置き・補足・カッコ書きは削り、補足は `explanation` へ。
   - 例: 「DNAという略称のもとになっている遺伝子の本体の正式な物質名は？」→「DNAの正式な物質名は？」。
6. **保育士試験の範囲外・一般的でない用語は出さない。** 救えなければ削除。**削除しても id は振り直さない**（歯抜けのまま。番号参照を安定させる）。
7. **1問1論点・テキストだけで解ける・年号や数値は検算。** 図/資料依存なら必要な数値・条件を本文に明記。

## A. フラッシュカード／一問一答（front=答え / back=問い）

> 本プロジェクトでは○×一問一答（qas.md の問い・補足）の確認に援用する。

### A-1. 削除する（修正で救えないもの）
- **手順・操作・説明文を答えにしている**：front が用語・値・現象でなく操作手順や説明文。
- **答えが一意に定まらない**：記述に複数の答えが当てはまる。修正で一意化できなければ削除。
- **例を答えさせる**：back が「〜の例は？」「代表例は？」、または front が例の列挙・末尾「など」。
- **あいまいな基準を答えにする**：漠然とした語句。
- **資格試験で扱わない応用・概念**：出題範囲外の前提を必要とする問題。

> 削除時は対象 id を取り除くだけで、**残りの id は振り直さない（歯抜けのまま）**。

### A-2. 修正する
- **問いと答えが噛み合っていない**：問いの言い換えになっている、または「なぜ？/どうなって？」に対し理由・結果になっていない。
  例:「果実ができないのはなぜ？」→ 答えは「子房がないから」（×「果実ができない」）。
- **問い（back）内のカッコ書きは原則削除**（補足・別名・言い換え）。情報は補足（補足説明 / `explanation`）へ。
- **「約／およそ」の整合**：前提が厳密な仮定なら答えに「約/およそ」は不要。前提自体に「約」があれば答えも「約」のまま。
- **一般的でない表現を平易に**。
- **不自然な言い回しを直す**：
  - 「〜を答えると？」→「〜は？」「〜を3つあげると？」など自然に。
  - 他動詞/自動詞の誤りを正す。
  - 余分・紛らわしい情報を削る。
- **難しい用語は補足に軽く説明**。
- **答えの粒度をそろえる**：抽象表現より具体名をメインに（抽象説明は補足へ）。
- front の難読漢字にはルビ `{漢字|よみ}`（読みは指定があればそれに従う）。

## B. クイズ（4択）

### B-1. 選択肢の作り（最重要）
- **正答だけ長く・詳しく・具体的にしない**。4つの選択肢の**長さ・粒度・構造をそろえる**。
  - 正答が突出して長いとき：誤答を同じ構造・長さの「もっともらしい誤り」に充実させる、または正答を意味を保って簡潔化する。
  - 順序問題は「同じステップの順序違い」、条件問題は「同じ要素の組合せ違い」、対象が固有名なら**4つとも固有名**にそろえる（例: 制度名を問うなら全部制度名、人物なら全部人名、年号なら全部年号）。
- **誤答は同じ分野内のもっともらしい誤りにする**。別分野の語・絶対にありえない選択肢・露骨に変な選択肢は禁止。
- **選択肢内のカッコ書きは原則削除**（補足・言い換え）。ただし次は残す：略語の定義（例「IQ（知能指数）」）、記号の説明、数式の括弧、意味の特定に必要な値。
  （`gen_4choice.py` が選択肢・qa.md の答えからカッコ書きを自動除去する。`SKILL.md`「選択肢のかっこ書き禁止」参照。）
- **「だけ」「のみ」を選択肢に入れない**。
- **不自然・場違いな選択肢を入れない**（文脈に合わない語。代わりに自然な誤りを使う）。
- 重複選択肢禁止。`correctIndex` は範囲内（0〜3）。

### B-2. 問題文
- **問題文から正答が容易に予測できる表現を避ける**（問題文に答えの語がそのまま入っている等）。語を言い換えるか、用途・性質・年代で選ばせる形にする。
- **手順・操作だけを問う設問は削除対象**になりやすい（価値が低ければ削除）。
- 図・資料・年表に依存する問題は、必要な数値・条件を問題文に明記してテキストだけで解ける形に。
- 年号・数値・計算は**検算**してから出す。

### B-3. その他
- 難易度（basic/standard/advanced）は維持。`explanation` は修正内容と整合させる。
- quiz を削除する場合も **id は振り直さない**。

## 適用後の自己チェック（必須）
- JSON/TSX 破損なし、`question`/`choices`/`correctIndex`/`explanation`（／qas.md の問題・答え・補足）以外のフィールド不変。
- 4択：選択肢4つ・重複なし・`correctIndex` 範囲内（0〜3）、「だけ/のみ」不使用。
- 長さ突出（正答が誤答平均の約1.7倍以上かつ+8字以上）が解消。
- 問い：手順・例・あいまい・噛み合わない・一意でない が残っていない。
- 問題文・正答の文言を変えた場合は、該当音声の再録要否を `SKILL.md` の表で確認した。
