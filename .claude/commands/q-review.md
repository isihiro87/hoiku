# 問題チェック

qas.mdファイルの問題を以下の観点でチェックし、**よりわかりやすい問題文を積極的に提案**します。

**レビュー方針**: 「問題がないからOK」ではなく、「もっとわかりやすくできないか？」という視点で全問題を見直す。

## 引数

- `$ARGUMENTS`: quizフォルダパス（例: `datas/history/grade2/4-1/3nanban_trade/quiz`）

---

## ステップ1: ファイルの読み込み

### 1.1 qas.mdファイルの検索

```bash
ls [フォルダパス]/*qas*.md
```

### 1.2 全ファイルの読み込み

各ファイルを読み込み、問題・答え・補足を抽出。

**qas.mdの構造**:
```
問題1
答え1
補足1

問題2
答え2
補足2
```

---

## ステップ2: 問題の改善チェック

各問題を以下の観点で**積極的に改善案を検討**:

### 重点チェック項目（必須）

1. **答えが一意に定まるか**
   - NG: 「織田信長の政策は？」→ 複数回答が可能
   - OK: 「織田信長が楽市楽座を行った城下町は？」→「安土」と一意に定まる

2. **中学生がぱっと読んで理解できるか**
   - 難しい熟語や専門用語を使いすぎていないか
   - 一文が長すぎないか（目安: 35文字以内）
   - 主語・述語が明確か

3. **補足が史実に沿っているか**
   - 歴史的事実と一致しているか確認
   - 年号、人物、出来事の関係が正確か

4. **あいまいなことを断定していないか**
   - 諸説ある事柄を断定的に書いていないか
   - 主観的な表現（「善意の」「偉大な」など）を避ける
   - 客観的な事実に基づいた記述にする

5. **学年にふさわしいやさしさの表現か**
   - フォルダパスから学年を判定（grade1=中1, grade2=中2, grade3=中3）
   - 学年が低いほど、よりやさしく平易な表現を心がける

   **中1（最もやさしく）**:
   - 難しい漢字はひらがなに（例: 跨る→またがる、佇まい→たたずまい）
   - 硬い熟語はかみ砕く（例: 継嗣→後継ぎ、策定→定めた）
   - 「〜における」「〜に際し」など硬い表現を避ける
   - 問題文は短めに（目安: 30文字以内）

   **中2（基本的な歴史用語OK）**:
   - 教科書に出る歴史用語はそのまま使える
   - ただし専門的すぎる表現は平易に（例: 布告→命令）
   - 問題文の目安: 35文字以内

   **中3（中学範囲の用語は自由に）**:
   - 中学で学ぶ範囲の用語は自由に使える
   - 高校レベルの専門用語は避ける
   - 問題文の目安: 35文字以内

### 問題文のスタイル（リズム重視）

**基本ルール**:
- **1問目**: 丁寧な質問文（「〜を何という？」「〜の名前は？」など）
- **2問目**: 必要に応じて丁寧に
- **3問目以降**: リズム重視で簡潔に（「〜は？」）

**スタイル例**:
| 問題番号 | スタイル | 例 |
|---------|---------|-----|
| 1問目 | 丁寧 | 水野忠邦が行ったのは何という改革？ |
| 2問目 | 丁寧/簡潔 | アメリカ商船が砲撃された事件の名前は？ |
| 3問目〜 | 簡潔 | 蘭学者が処罰された事件は？ |
| 4問目〜 | 簡潔 | 1837年に大阪で起きた反乱は？ |

**使える表現パターン**:
- 「何〇〇？」: 答えが「△△〇〇」の形式の場合（何幕府？→江戸幕府）
- 「〜を何という？」: 用語を問う場合（1問目・2問目向け）
- 「〜の名前は？」: 事件名・人名を問う場合（1問目・2問目向け）
- 「〜は？」: 簡潔に問う場合（3問目以降向け）
- 「どこ？」「だれ？」: 場所・人物を問う場合

**避けるべき表現**:
- 「武家政権」「区分」「総称」など中学生に難しい言葉
- 回りくどい表現（「〜させたのは？」など）
- 「善意の」「偉大な」など主観的な修飾語

### 補足説明のスタイル

- **言い切り形式を使う**（敬語「〜です。」「〜ます。」は使わない）
- **史実に基づく客観的な記述**にする
- 簡潔で読みやすい表現にする

**修正パターン例**:
| 修正前 | 修正後 |
|--------|--------|
| 善意の船を撃退した。 | 武装していない船を砲撃してしまった。 |
| この敗北を知った水野忠邦は〜 | この結果を知り、幕府は〜 |
| 捕らえられ自ら命を絶った。 | のちに脱獄するも自害した。（史実に合わせて修正） |

### 改行ルールの確認（レンダリング時に重要）

- 問題文に `\n` を入れる場合、**各行は11文字以内**（12文字以上は禁止）
- 文字数の数え方:
  - 日本語（ひらがな、カタカナ、漢字、句読点、括弧）: 1文字 = 1
  - 半角英数字: 2文字 = 1（例: 1603 = 2文字分）
- 11文字を超える場合は、別の位置で区切るか、改行なしで自動改行に任せる

---

## ステップ3: 改善案の実施

チェック結果に基づき、**全問題に対して改善を検討**:

- **改善案がある問題**: より分かりやすい表現に修正
- **すでに最適な問題**: そのまま保持（ただし「最適である理由」を意識する）

**積極的に改善すべきポイント**:
- 答えが一意に定まらない → 条件を追加して特定
- 難しい言葉がある → 中学生にわかりやすい言葉に置き換え
- 学年に対して表現が硬い → よりやさしい表現に（特に中1）
- 長い問題文 → 簡潔に言い換え
- 補足が史実と異なる → 正確な記述に修正
- 補足が主観的 → 客観的な事実に修正
- 補足が敬語になっている → 言い切り形式に変更

---

## ステップ4: ファイル出力

### 4.1 修正済みqas.mdの生成

修正を適用したqas.mdファイルを生成。

**保存先**: `[フォルダパス]/qas_reviewed.md`

**フォーマット**:
```
修正後の問題1
答え1
補足1

修正後の問題2
答え2
補足2
```

※ 改善の余地がない問題は元のまま含める。

### 4.2 レビュー記録の生成

全問題の検討結果と修正内容を記録。

**保存先**: `[フォルダパス]/qas_review.md`

**フォーマット**:
```markdown
# 問題レビュー記録

生成日: [日付]
対象: [元ファイル名]

## サマリー

- 全問題数: [N]問
- 修正なし: [X]問
- 修正あり: [Y]問

## 方針

- 1問目: 丁寧な質問文（「〜を何という？」など）
- 2問目: 必要に応じて丁寧に
- 3問目以降: リズム重視で簡潔に（「〜は？」）

---

## 検討内容

### Q[番号]: [修正あり/現状維持]

**元の問題文**:
[元の問題文]

**改善後**:（修正ありの場合）
[修正後の問題文]

**検討内容**:
[改善した理由、または現状が最適と判断した理由]

**補足の修正**:（補足を修正した場合）
[修正内容と理由]

---
```

※ 修正がなかった問題も「検討したが現状が最適」と判断した理由を簡潔に記載。
